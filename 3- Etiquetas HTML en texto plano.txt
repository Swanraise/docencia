*WINDOW EVENT ATTRIBUTES*
1- onafterprint: Te avisa de que has empezado a imprimir algo (cuando accedes Ctrl + P, aunque luego le hayas dado a cancelar ¬¬).
2- onbeforeprint: Salta un aviso de que, efectivamente, vas a imprimir este documento (también depende del navegador soportado.
3- onbeforeunload: Si cierras la ventana, pulsas F5 o vas a cambiar tu URL, sale un aviso en plan "Tus cambios no se guardarán, ¿estás seguro de salir?".
4- onerror: Ventana emergente y aviso de que hay algo que no se ha podido cargar en la página.
5- onhashchange: Sucede cuando han habido cambios en la parte anchor [CSS] (empieza con el símbolo '#') de la URL actual.
6- onload: Aviso emergente de que la página ya ha sido por fin cargada.
7- onmessage: Cuando se activa un mensaje aparecerá otro mensaje impreso (mostrado) en pantalla.
8- onoffline: El código aparecerá si el navegador empieza a trabajar fuera de línea.
9- ononline: El código aparecerá si el navegador empieza a trabajar en línea.
10- onpagehide: Código que se ejecutará cuando el usuario navegue alejándose de una página.
11- onpageshow: Ventana emergente cuando el usuario navega a una página web.
12- onpopstate: Mensaje que surge cuando el historial cambia.
13- onresize: Al modificar el tamaño de ventana aparecerá un mensaje. Cada vez que hagas la más mínima modificación.
14- onstorage: Mensaje de "La tienda de una página web" ha sido actualizada, en ese plan.
15- onunload: Cuando dejas de cargar cierta página o se cierra el buscador, aparecerá dicho mensaje.

*FORM EVENTS*
1- onblur: Cuando escribes en un input, formatea su estilo temporalmente (texto y background) mientras escribes dentro.
2- onchange: Seleccionar un elemento de una lista desplegable que indicará lo que hayas escogido.
3- oncontextmenu: Menú contextual que surge cuando haces click derecho sobre un elemento concreto para ver más opciones.
4- onfocus: Cuando haces click en un elemento, cambia su formato inmediatamente por mucho que lo descliques después.
5- oninput: Sale duplicado a tiempo real lo que estés escribiendo en el elemento input o textarea en otro lado de la pantalla.
6- oninvalid: Cuando envías algo inválido dentro de un elemento input.
7- onreset: Resetear algo escrito y borrarlo de golpe al clicar un botón destinado a tal efecto, por ejemplo.
8- onsearch: Escribes algo, le das a Enter y te sale un elemento de tipo search indicando la búsqueda que estás efectuando.
9- onselect: Cuando seleccionas parte de un texto con el botón primario del ratón saldrá un mensaje emergente (ideal para evitar copy-past en tu web).
10- onsubmit: Reitera lo que efectivamente hayas subido al darle al input.

*KEYBOARD EVENTS*
1- onkeydown: Mientras pulsas una tecla dentro del input saldrá una ventana emergente.
2- onkeypress: Cuando hayas pulsado una tecla dentro del input saldrá un pop-up emergente.
3- onkeyup: Cuando sueltes o dejes de apretar una tecla dentro del input saldrá el aviso.

*MOUSE EVENTS*
1- onclick: Si haces click con el ratón disparas un evento del elemento accionado.
2- ondblclick: Si haces doble click seguido con el ratón disparas un evento del elemento accionado.
3- onmousedown: Cuando pulsas con el botón primario del ratón sobre un texto, puedes hacerlo cambiar de color durante y tras el clicado.
4- onmousemove: Cuando posas el ratón encima de una imagen, ésta se agrandará automáticamente (como en las tiendas Amazon).
5- onmouseout: Parecido pero en teoría cuando mueves el puntero fuera de un elemento.
6- onmouseover: Parecido pero cuando el puntero se mueve sobre un elemento (frecuentemente una imagen).
7- onmouseup: Cuando pulsas con el botón primario del ratón sobre un texto, puedes hacerlo cambiar de color durante y tras el clicado.
8- onwheel: Si activas la ruleta del ratón encima de un elemento, realizará una acción, como agrandarse por ejemplo.

*DRAG EVENTS*
1- ondrag: Un evento ocurre si está siendo arrastrado un texto o un elemento.
2- ondragend: Un evento que ocurre si un texto o un elemento ha sido arrastrado de un punto A a un punto B.
3- ondragenter: El arrastre permite indicarte si al querer depositarlo en un lugar (mientras mantienes preto el click izqdo) entraría o no dicho elemento.
4- ondragleave: Lo mismo pero una vez has soltado el elemento arrastrado de una zona A a una zona B.
5- ondragover: Cuando un elemento o texto arrastrable está siendo arrastrado sobre un objetivo de depósito válido.
6- ondragstart: Cuando comienzas a arrastrar un elemento o texto de un punto A a un punto B sin resaltar las áreas involucradas en dicha operación.
7- ondrop: Igual que el anterior pero una vez ya has arrastrado el elemento, no mientras lo estás arrastrando o comenzado a arrastrar.
8- onscroll: Si haces ruleta en cierta zona o texto, no solo avanzará si hay información todavía por mostrarse sino que además puedes formatear su estilo.

*CLIPBOARD EVENTS*
1- oncopy: Te avisa cuando has copiado un texto.
2- oncut: Te avisa cuando has cortado (que no copiado) un texto.
3- onpaste: Te avisa cuando has pegado un texto.

*MEDIA EVENTS*
1- onabort: Script que se ejecutará al cancelar.
2- oncanplay: Secuencia de comandos que se ejecutará cuando un archivo esté listo para comenzar a reproducirse (cuando se haya almacenado lo suficiente para comenzar).
3- oncanplaythrough: Secuencia de comandos que se ejecutará cuando un archivo se pueda reproducir hasta el final sin pausa para almacenar.
4- oncuechange: Script que se ejecutará cuando la cue cambie en un elemento track.
5- ondurationchange: Script que se ejecutará cuando cambie la longitud del medio.
6- onemptied: Secuencia de comandos que se ejecutará cuando ocurra algo malo y el archivo de repente no esté disponible (como si se desconectara inesperadamente).
7- onended: Secuencia de comandos que se ejecutará cuando los medios hayan llegado al final (un evento útil para mensajes como "gracias por escuchar").
8- onerror: Script que se ejecutará cuando se produzca un error al cargar el archivo.
9- onloadeddata: Secuencia de comandos que se ejecutará cuando se carguen datos multimedia.
10- onloadedmetadata: Script to be run when meta data (like dimensions and duration) are loaded.
11- onloadstart: Secuencia de comandos que se ejecutará justo cuando el archivo comience a cargarse antes de que se cargue nada.
12- onpause: Secuencia de comandos que se ejecutará cuando el medio esté en pausa, ya sea por el usuario o mediante programación.
13- onplay: Secuencia de comandos que se ejecutará cuando el medio esté listo para comenzar a reproducirse.
14- onplaying: Secuencia de comandos que se ejecutará cuando el medio haya comenzado a reproducirse.
15- onprogress: Secuencia de comandos que se ejecutará cuando el navegador esté en proceso de obtener los datos multimedia.
16- onratechange: Secuencia de comandos que se ejecutará cada vez que cambie la velocidad de reproducción (como cuando un usuario cambia a cámara lenta o modo de avance rápido).
17- onseeked: Secuencia de comandos que se ejecutará cuando el atributo de búsqueda se establezca en falso, lo que indica que la búsqueda ha finalizado.
18- onseeking: Secuencia de comandos que se ejecutará cuando el atributo de búsqueda se establezca en verdadero, lo que indica que la búsqueda está activa.
19- onstalled: Secuencia de comandos que se ejecutará cuando el navegador no pueda obtener los datos multimedia por cualquier motivo.
20- onsuspend: La secuencia de comandos que se ejecutará cuando la obtención de datos multimedia se detiene antes de que se cargue por completo por cualquier motivo.
21- ontimeupdate: Secuencia de comandos que se ejecutará cuando la posición de reproducción haya cambiado (como cuando el usuario avanza rápidamente a un punto diferente en el medio).
22- onvolumechange: Secuencia de comandos que se ejecutará cada vez que se cambie el volumen, lo cual (incluye configurar el volumen en "silencio").
23- onwaiting: Secuencia de comandos que se ejecutará cuando el medio se haya detenido pero se espera que se reanude (como cuando el medio se detiene para almacenar más datos en el búfer).

11. pause: Se dispara cuando el audio / video se ha detenido.
12. play: Se activa cuando el audio / video se ha iniciado o ya no está en pausa.
13. playing: Se dispara cuando el audio / video se está reproduciendo después de haber sido pausado o detenido para almacenamiento en búfer.
14. progress: Se activa cuando el navegador está descargando el audio / video.
15. ratechange: Se dispara cuando se cambia la velocidad de reproducción del audio / video.
16. seeked: Se activa cuando el usuario termina de moverse / saltar a una nueva posición en el audio / video.
17. seeking: Se dispara cuando el usuario comienza a moverse / saltar a una nueva posición en el audio / video.
18. stalled: Se activa cuando el navegador intenta obtener datos multimedia, pero los datos no están disponibles.
19. suspend: Se activa cuando el navegador no obtiene datos multimedia de forma intencionada.
20. timeupdate: Se dispara cuando la posición de reproducción actual ha cambiado.
21. volumechange: Se dispara cuando se cambia el volumen.
22. waiting: Se dispara cuando el video se detiene porque necesita almacenar en búfer el siguiente cuadro.
